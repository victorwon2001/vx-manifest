# 작업 규칙

- 한국어로 작업 로그와 문서를 작성한다.
- 공개 문서에는 내부 운영 맥락이나 특정 외부 서비스명을 직접 노출하지 않는다.
- 모듈 추가 또는 개선 시 `tools/release.ps1`로 버전, changelog, commit, push를 함께 처리한다.
- 공통 진입은 로더가 맡고, 기능은 `modules/<moduleId>/main.js` 단위로 분리한다.
- 각 PC의 활성화 상태는 로컬 저장소 기준으로 관리한다.
- 새 모듈을 추가할 때는 먼저 사용자에게 표시 이름을 확인받고, 그 이름을 `config/registry.json`과 `modules/<moduleId>/meta.json`에 반영한 뒤 업로드한다.
- 대상 서비스에 직접 접근했는데 권한이 없거나 로그아웃 상태면 먼저 로그인한 뒤 다시 접근한다.
- 사용자가 특정 링크를 준 경우에는 로그인 후 그 링크로 바로 재접속한다.
- 로그인 아이디와 비밀번호 같은 민감 정보는 저장소 문서나 공개 파일에 적지 않는다.
- 로그인 정보 같은 로컬 민감값은 저장소 루트의 `.env.local`에서 관리하고, 로그인 필요 작업에서는 해당 값을 우선 사용한다.
